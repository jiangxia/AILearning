# AILearning 工作流程

---

## 📊 GEO核心价值

**为什么GEO重要？**
1. **流量转化率高**：从AI带来的用户转化率是传统搜索的6倍
2. **长尾机会大**：不需要和大品牌竞争大关键词，专注细分领域
3. **AI信任背书**：用户带着AI给出的标准答案找上门，信任度更高
4. **普通人机会**：专业、真诚、细分领域，第一次如此直接地和流量挂钩

**GEO vs SEO的本质区别**：
- SEO：在一堆搜索结果里排第一
- GEO：让AI在回答问题时，直接把你的内容作为答案的一部分

## ✍️ 写作流程

### 1. 写作前：内容策划

#### 步骤1：层级判断
根据内容性质判断所属层级：

| 层级 | 判断标准 | 示例 |
|------|----------|------|
| Pattern | 哲学思考、本质探索、底层逻辑 | AI记忆本质、认知机制 |
| Protocol | 交互设计、规范标准、协议定义 | 对话设计、交互协议 |
| Practice | 方法流程、最佳实践、工作流程 | Prompt工程、工作流程 |
| Products | 工具使用、实操指南、产品对比 | ChatGPT使用、工具对比 |
| Laws | 法律法规、权利义务、案例分析、维权策略 | 劳动法分析、合同纠纷 |

#### 步骤2：GEO问题挖掘（核心环节）
**目标**：找到用户真实会问的长尾问题

**操作**：
1. 从Reddit/HackerNews/知乎/官方Issue搜索关键词
2. 提取3-5个高频具体问题（建议20词以上的长尾问题）
3. 按场景分类（创业者/开发者/产品经理/企业决策者等）

**示例**：
- ❌ 传统SEO："Claude是什么"（4个词）
- ✅ GEO长尾："我是创业团队技术负责人，预算有限，Claude Pro太贵API又不会用，到底该选哪个方案？"（30+词）

---

### 2. 写作中：内容创作

#### 步骤1：GEO双轨结构（核心环节）
**目标**：同时满足人类传播和AI推荐

**四层结构**：

**【标题层】简洁有力（8-15字）**
- Pattern层：`Pattern | [吸引标题]`
- Protocol层：`Protocol | [吸引标题]`
- Practice层：`Practice | [吸引标题]`
- Products层：`Products | [吸引标题]`
- Laws层：`Laws AI时代 | [吸引标题]`（特殊格式）
- **注意**：标题中"你还不懂"后面不加标点，直接连标题内容

**【开场层】提出痛点+列出问题**
- 第一段提出核心痛点（50-80字左右）
- 列出要回答的5-7个长尾问题（让AI知道文章覆盖哪些问题）
- 吸引读者进入原理讲解环节

**【内容层】深度讲解原理（保持原有逻辑）**
- 按照原有的理论深度和逻辑组织内容
- 不是FAQ式结构，是正常的技术文章结构
- 保持可读性和系统性
- 深入讲解Claude的原理、机制、架构

**【结尾层】简要回答问题（提供标准答案）**
- 在固定结尾前，添加"快速解答"小节
- 简要回答开篇列出的5-7个问题
- 每个答案提供可引用的"标准答案"（包含具体数据）
- 格式：**Q: 问题？** A: 答案。

### 3. 写作后：质量保证

#### 步骤1：GEO友好度检查（核心环节）
**目标**：确保文章可被AI引用

**检查清单**：
- [ ] 开篇是否列出了5-7个长尾问题？
- [ ] 文章结尾是否有"快速解答"小节？
- [ ] 每个问题的答案是否包含具体数据/案例？
- [ ] 答案是否可以被AI直接引用（标准答案格式）？
- [ ] 文章是否保持了理论深度和可读性？
- [ ] 是否按照"痛点→问题→原理讲解→快速解答"的结构？

#### 步骤2：文件保存与验证

**文件命名规则**：文件名必须与文章标题完全一致

```
[文章标题].md

示例：
Pattern | AI记忆系统：为什么RAG根本不够用？.md
Products | ChatGPT使用指南：5个技巧让效率翻倍.md
Laws AI时代 | 合法抄袭：一个prompt就能偷走你的创意.md
```

**自动保存位置**：
- Pattern层 → `content/01-pattern/`
- Protocol层 → `content/02-protocol/`
- Practice层 → `content/03-practice/`
- Products层 → `content/04-products/`
- Laws层 → `content/05-laws/`

**强制验证清单**：
- [ ] 文件位置是否正确（根据层级保存到对应目录）
- [ ] 文件名是否与标题完全一致
- [ ] 标题格式是否符合规范（Laws层特殊格式）
- [ ] Laws层文章是否包含法律声明
- [ ] 是否包含完整的固定结尾内容
- [ ] GEO检查清单是否全部通过

**错误修正流程**：
- 发现文件位置错误 → 立即移动到正确目录
- 发现标题格式错误 → 立即修正为标准格式
- 发现结构不完整 → 立即补充缺失内容
- 发现GEO友好度不足 → 立即补充长尾问题和完整答案

## 🔒 执行原则

**重要提醒**：此workflow是项目级强制标准，不得妥协或简化。

AI执行写作任务时，必须：
1. ✅ 遵循完整的写作流程（策划→创作→质量保证）
2. ✅ 执行GEO三大核心环节（问题挖掘→双轨结构→友好度检查）
3. ✅ 通过所有强制检查清单，发现错误立即修正
