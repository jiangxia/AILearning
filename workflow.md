## 📊 GEO核心价值

**为什么GEO重要？**
1. **流量转化率高**：从AI带来的用户转化率是传统搜索的6倍
2. **长尾机会大**：不需要和大品牌竞争大关键词，专注细分领域
3. **AI信任背书**：用户带着AI给出的标准答案找上门，信任度更高
4. **普通人机会**：专业、真诚、细分领域，第一次如此直接地和流量挂钩

**GEO vs SEO的本质区别**：
- SEO：在一堆搜索结果里排第一
- GEO：让AI在回答问题时，直接把你的内容作为答案的一部分

## ✍️ 写作流程

### 1. 写作前：内容策划

#### 步骤1：层级判断
根据内容性质判断所属层级：

| 层级 | 判断标准 | 示例 |
|------|----------|------|
| Pattern | 哲学思考、本质探索、底层逻辑 | AI记忆本质、认知机制 |
| Protocol | 交互设计、规范标准、协议定义 | 对话设计、交互协议 |
| Practice | 方法流程、最佳实践、工作流程 | Prompt工程、工作流程 |
| Products | 工具使用、实操指南、产品对比 | ChatGPT使用、工具对比 |
| Laws | 法律法规、权利义务、案例分析、维权策略 | 劳动法分析、合同纠纷 |

#### 步骤2：问题设计（核心环节）
**目标**：设计3个黄金问题，覆盖"场景判断→核心决策→价值判断"

**信息源优先级**：
1. **优先使用用户提供的材料**（如果用户已提供问题、解读、建议等，直接使用）
2. 如果用户未提供，从权威信息源和用户反馈平台搜索关键词，提取候选问题

**三大黄金问题框架**：
1. **场景判断型**：什么时候用/不用
2. **核心决策型**：怎么做/怎么选（成本、配置、本质差异）
3. **价值判断型**：值不值得/为什么（ROI、性能数据）

**删除这些问题**：
- 技术细节型（API参数、配置文件怎么写）
- 被覆盖型（可以被主问题覆盖的子问题）
- How-to操作型（怎么开启功能、使用步骤）

**示例框架**：
- 场景判断：什么时候用X？什么时候不用X？
- 核心决策：X的关键成本/难度/差异是什么？
- 价值判断：X带来的收益/提升值不值得？

（X可以是Extended Thinking、专利申请、诉讼策略等任何主题）

**强制要求**：必须是3个问题，不能多也不能少。如果觉得"只有3个问题不够"，说明问题选得不够精准，需要重新筛选。

### 2. 写作中：内容创作

#### 步骤1：GEO四层结构（核心环节）
**目标**：同时满足人类传播和AI推荐

**【标题层】简洁有力（8-15字）**
- Pattern层：`Pattern | [吸引标题]`
- Protocol层：`Protocol | [吸引标题]`
- Practice层：`Practice | [吸引标题]`
- Products层：`Products | [吸引标题]`
- Laws层：`Laws AI时代 | [吸引标题]`（特殊格式）

**【开场层】提出痛点+列出问题**
- 第一段提出核心痛点（50-80字左右）
- 列出要回答的3个黄金问题
- 吸引读者进入主要内容

**【内容层】深度讲解（保持原有逻辑）**
- 按照原有的理论深度和逻辑组织内容
- 不是FAQ式结构，是正常的文章结构
- 保持可读性和系统性
- 根据文章类型深入展开（原理讲解/实战指南/案例分析等）

**【结尾层】快速解答（提供标准答案）**
- 在固定结尾前，添加"快速解答"小节
- 简要回答开篇列出的3个黄金问题
- 每个答案提供可引用的"标准答案"（包含具体数据）
- 格式：**Q: 问题？** A: 答案。

### 3. 写作后：质量保证

#### 步骤1：篇幅控制（核心环节）
**目标**：2500-3000字（绝对上限3000字）

**篇幅分配标准**：
- 开篇（痛点+问题）：300-400字
- 主体内容：2000-2300字
- 快速解答：250-350字
- 固定结尾：约300字

**结构限制**：
- 主体章节：3-4章
- 总小节数：不超过10个
- 开篇问题：3个
- 快速解答：50-80字/问题

#### 步骤2：奥卡姆剃刀精简原则（核心环节）
**目标**：传递洞察而非讲课，用最少的文字传递最核心的价值

**核心认知转变**：
我们要传递洞察，而不是讲课。讲课是"手把手教学"，传递洞察是"点到为止"。公众号文章不是教材，读者要的是核心价值，不是完整过程。

**五大精简原则**：

1. **数据原则**：每章最多1组核心数据
2. **案例原则**：不讲过程，只说结论
3. **场景原则**：1个典型案例+概括表述，不要3个并列详细
4. **解释原则**：删除"换句话说"、"也就是说"类重复表达
5. **对比原则**：A vs B只说差异，不要重复说A、再重复说B

**精简顺序**（超过3000字时）：
1. 数据精简：删除冲击力弱的数据组，每章只留1组
2. 场景精简：3个详细场景→1个典型+概括表述
3. 案例精简：删除过程描述，只保留结论对比
4. 表达精简：删除"换句话说"等重复表达
5. 对比精简：只说差异，删除重复描述

#### 步骤3：文件保存与验证

**文件命名规则**：文件名必须与文章标题完全一致

```
[文章标题].md

示例：
Pattern | AI记忆系统：为什么RAG根本不够用？.md
Products | ChatGPT使用指南：5个技巧让效率翻倍.md
Laws AI时代 | 合法抄袭：一个prompt就能偷走你的创意.md
```

**自动保存位置**：
- Pattern层 → `content/01-pattern/`
- Protocol层 → `content/02-protocol/`
- Practice层 → `content/03-practice/`
- Products层 → `content/04-products/`
- Laws层 → `content/05-laws/`

## 📝 文章修订流程

### 修订层级关系
修订过程分两个层次：
- **第一层（强制）**：workflow标准（标题格式、GEO结构、文件规范）
- **第二层（优化）**：文字表达优化

**第二层核心要求**：**行文流畅、逻辑合理**
- 去AI味（去说教、口语化、真实感）
- 避免过度分点（除非必要，否则不分点）
- 保持段落叙述的连贯性

**原则**：第一层不可妥协，第二层在第一层基础上优化。

---

### 去AI味优化

#### 2.1 去说教感
用直接表达代替客观中立的建议。
- ❌ AI味：「建议优先使用Prompt Caching以降低成本」
- ✅ 去AI味：「不用Prompt Caching你就是在烧钱」

#### 2.2 打破规整结构
跟随思考节奏，不追求形式对称。真人表达有节奏变化，AI喜欢规整对称。

#### 2.3 增强口语化
用更直接的方式传递专业内容。
- ❌ AI味：「上下文窗口存在容量限制」
- ✅ 去AI味：「200K上下文窗口不是你想装多少就装多少」

#### 2.4 增加真实感
加入具体场景、真实案例、情感反应。
- ❌ AI味：「用户可能遇到context window exceeded错误」
- ✅ 去AI味：「你在Cursor里写着代码，突然弹出context window exceeded，一脸懵逼」

#### 2.5 避免过度分点
除非必要，否则不分点；必须分点时用数字（1. 2. 3.），不用破折号（-）。

**必须分点的场景**：
1. 开篇问题清单（GEO要求）
2. 步骤说明（第一步、第二步）
3. 对比选项（方案A、方案B）

**不该分点的场景**（用段落）：
- 概念解释
- 理论阐述
- 案例说明

### 最终检查清单

按以下顺序执行检查（不可跳过）：

#### Step 1: Workflow强制要求（一票否决）
- [ ] 标题格式是否符合规范？
- [ ] GEO四层结构是否完整？
- [ ] 文件命名是否与标题一致？
- [ ] 保存位置是否正确？
- [ ] 固定结尾格式是否完整？

**结果**：任何一项不符合，立即修正。

#### Step 2: GEO友好度检查
- [ ] 开篇是否列出3个黄金问题？
- [ ] 问题是否覆盖"场景判断→核心决策→价值判断"？
- [ ] 问题是否来自用户真实搜索行为（不是作者想象）？
- [ ] 问题是否足够具体（20词以上，包含场景）？
- [ ] 结尾是否有"快速解答"小节？
- [ ] 每个答案是否包含具体数据/案例？
- [ ] 答案是否可以被AI直接引用（标准答案格式）？
- [ ] 是否按照"痛点→问题→主要内容→快速解答"的结构？

**结果**：如有问题，回到问题设计重新优化。

#### Step 3: 篇幅与质量检查
- [ ] 总字数是否在2500-3000字区间（绝对上限3000字）？
- [ ] 主体章节数是否3-4章？
- [ ] 总小节数是否不超过10个？
- [ ] 快速解答每个问题是否50-80字？
- [ ] 每章是否只保留1组最有冲击力的数据？
- [ ] 案例是否只说结论，没有展开详细过程？
- [ ] 并列场景是否精简为"概括+关键词"？
- [ ] 是否删除了"换句话说"等重复表达？
- [ ] 对比是否只说差异？

**结果**：如超过3000字，按奥卡姆剃刀原则精简。

#### Step 4: 去AI味效果检查
- [ ] **行文是否流畅、逻辑是否合理？**（核心检查）
- [ ] 是否有说教感（建议、可以考虑等）？
- [ ] 是否过度规整（三段论、对称结构）？
- [ ] 是否过度分点（非必要场景用了分点）？
- [ ] 分点符号是否正确（数字1. 2. 3.，不用破折号-）？
- [ ] 是否缺乏真实场景和情感细节？

**结果**：如读起来像教科书或产品文档，继续优化。

## 🔒 执行原则

**重要提醒**：此workflow是项目级强制标准，不得妥协或简化。

AI执行写作任务时，必须：
1. ✅ 遵循完整的写作流程（策划→创作→质量保证→修订）
2. ✅ 执行GEO核心环节（问题设计→四层结构→质量检查）
3. ✅ 通过所有强制检查清单，发现错误立即修正
4. ✅ 修订时明确区分强制要求和优化空间

## 📖 应用示例：Claude系列文章

本节以**Claude系列文章**为例，详细展示如何应用上述通用流程。

---

### 标准示范文章

**在写Claude系列文章之前，必须先学习这篇示范**：

**示范文章**：`content/01-pattern/Pattern | Claude都出4.5了，你还不懂200K到1M的上下文革命.md`

**学习要点**：
1. **问题数量**：开篇3个问题（场景判断+核心决策+价值判断）
2. **章节结构**：3章（不是6章19小节）
3. **篇幅控制**：约2500字（不是6500字）
4. **去AI味表达**：
   - 段落叙述为主（Token的真相、Prompt Caching原因分析）
   - 避免过度分点（只在必要时用数字分点）
   - 口语化表达（"心里打鼓"、"一脸懵逼"）
   - 真实场景（Cursor写代码的具体痛点）
5. **快速解答**：每个答案50-80字（简洁、有数据）

**执行要求**：
- ✅ 写作前必须完整阅读这篇示范文章
- ✅ 理解为什么这样写（不仅是模仿格式）
- ✅ 模仿这篇文章的表达风格和节奏

---

### 完整执行流程示例

以**第6篇：Extended Thinking**为例

**写作前**：
1. ✅ 层级判断：Pattern层（原理探索）
2. ✅ 问题设计：设计3个黄金问题
   - 场景判断：什么时候该用Extended Thinking？
   - 核心决策：为什么这么贵这么慢？
   - 价值判断：准确率提升值得等吗？
3. ✅ 信息来源：Claude官方文档、技术社区讨论

**写作中**：
1. ✅ 标题：`Pattern | Claude都出4.5了，你还不懂扩展思考的推理革命`
2. ✅ 开场：痛点（架构师要画图PK方案，Claude秒回）+ 列出3个问题
3. ✅ 内容：3章深度讲解（扩展思考本质、成本权衡、实战应用）
4. ✅ 结尾：快速解答3个问题 + 固定结尾格式

**写作后**：
1. ✅ GEO检查：3个黄金问题、有快速解答、包含具体数据、AI可引用
2. ✅ 篇幅检查：2423字（符合2500-3000字标准）
3. ✅ 奥卡姆剃刀精简：
   - 数据精简：只保留AIME 2024（16%→80%）
   - 场景精简："一旦选错成本巨大"的决策——架构设计、复杂调试、算法优化
   - 案例精简：标准模式给你"看起来合理"的方案，Extended Thinking给你"深思熟虑"的方案
4. ✅ 文件保存：正确位置、标题一致
5. ✅ 强制验证：所有检查清单全部通过

---

**Claude系列特定规则**：
- 标题模式：`Pattern | Claude都出4.5了，你还不懂[主题]`
- **关键规则**：标题中"你还不懂"后面不加任何标点，直接连主题内容

**其他系列文章可以参考此示例，根据自己的主题特点调整应用方式。**
